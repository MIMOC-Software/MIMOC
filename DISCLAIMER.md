# Operational Disclaimer

MIMOC — Modular IT Management Orchestrated Console — is an IT administration and orchestration tool.

It may perform high-impact operations on systems, networks and endpoints, including but not limited to:

- network discovery;
- WMI, WinRM, SSH and SNMP queries;
- software inventory and software scanning;
- Wake-on-LAN;
- reboot and shutdown operations;
- software deployment and removal;
- scheduled automation;
- credential storage and credential usage;
- database backup and restore operations.

## Authorized use only

MIMOC must be used only on systems, networks and endpoints for which you have explicit authorization.

You must not use MIMOC to scan, access, control, modify, monitor or administer systems or networks without permission.

The user or organization installing and operating MIMOC is responsible for ensuring that all usage complies with:

- applicable laws and regulations;
- internal IT policies;
- cybersecurity policies;
- privacy and data protection requirements;
- contractual obligations;
- employment and monitoring rules.

## Operational responsibility

The user or organization operating MIMOC is responsible for:

- validating the environment before installation;
- configuring permissions and credentials correctly;
- testing actions on limited target groups before broad execution;
- maintaining backups;
- verifying firewall, network and endpoint policies;
- managing user roles and access rights;
- reviewing audit logs;
- using maintenance mode and scheduling controls appropriately.

MIMOC is designed to help control operational risk, but it cannot replace administrative judgement, change management or authorization procedures.

## No automatic compliance guarantee

MIMOC does not automatically guarantee compliance with any legal, regulatory, security or industry standard.

Compliance depends on how the software is configured, deployed, governed and used within the organization.

## Sensitive data

Do not publish or share MIMOC logs, screenshots, exports or database files publicly if they contain sensitive information such as:

- usernames;
- hostnames;
- IP addresses;
- MAC addresses;
- software inventory;
- credentials;
- license files;
- audit logs;
- internal network details.

Before sharing diagnostic information, remove or anonymize sensitive data.
