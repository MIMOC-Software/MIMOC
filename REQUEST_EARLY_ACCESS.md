# Request Early Access to MIMOC

MIMOC Community / Early Access builds are distributed in a controlled way.

This helps ensure that users understand the operational scope of the software before installing it.

## Before requesting access

Please read:

- `README.md`
- `DISCLAIMER.md`
- `SECURITY.md`
- `INSTALLATION.md`

MIMOC is an IT administration tool and may perform high-impact operations such as network discovery, WMI/WinRM/SSH/SNMP queries, Wake-on-LAN, reboot, shutdown, deployment and scheduled automation.

Use MIMOC only on systems, networks and endpoints for which you have explicit authorization.

## How to request access

Open a GitHub issue in this repository with the subject:

```text
MIMOC Early Access request
```

Use the template below.

```text
Additional notes:
```

## What happens next

Access may be granted to a private Early Access repository or through a controlled download channel.

Early Access builds may include:

- installer package;
- SHA256 checksum;
- release notes;
- documentation;
- known limitations.

## Do not include sensitive data

Do not include any of the following in public GitHub issues:

- passwords;
- credentials;
- private keys;
- license files;
- internal hostnames;
- public or private IP ranges;
- logs with sensitive data;
- screenshots containing confidential infrastructure details;
- customer or personal data.

## Important

Submitting a request does not guarantee access.

Access may be limited, delayed or denied depending on project phase, testing capacity and operational risk.
